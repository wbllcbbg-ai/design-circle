# 设计圈功能头脑风暴

> 日期：2026-05-30
> 基于现有代码库分析，从六个维度系统性思考未来功能方向

---

## 一、用户粘性与互动

| 功能 | 描述 | 优先级 | 理由 |
|---|---|---|---|
| **点赞/收藏后端打通** | 把前端点赞、收藏、关注按钮接上 Supabase | 🔴 立即做 | 前端的按钮点不了，最基础的互动都不行 |
| **文章评论区** | 文章详情页加上跟案例一样的评论区 | 🔴 立即做 | 文章有内容但不能讨论，体验断裂 |
| **互动通知** | 有人评论/点赞/关注你时，右上角小红点通知 | 🟡 做完基础互动后做 | 没有通知就没有回来的动力 |
| **关注设计师** | 用户可以关注设计师，首页 Feed 优先展示关注的人 | 🟡 中优先级 | 建立粉丝链路，让设计师愿意持续发内容 |
| **浏览历史** | 记录用户看过的案例和文章，/profile 里可查看 | 🔵 低优先级 | 解决"之前看过的案例找不到了"的痛点 |
| **每日精选推送** | 每日推荐热门案例/文章 | 🔵 长期规划 | 需要内容量够大才有价值 |

## 二、设计师视角

| 功能 | 描述 | 优先级 | 理由 |
|---|---|---|---|
| **设计师工作台** | 专属后台：查看自己的案例、评价、咨询消息、数据统计 | 🔴 高优先级 | 激发设计师持续入驻的核心动力 |
| **我的案例管理** | 设计师可以增删改自己的案例 | 🟡 中优先级 | 现在发了就不能改 |
| **回复评价** | 设计师可以对用户的评价进行回复沟通 | 🟡 中优先级 | 促进设计师和用户互动 |
| **咨询消息打通** | 用户点"咨询设计师"→ 创建对话 → 设计师收到消息 | 🟡 中优先级 | 平台的核心连接价值 |
| **设计师排行榜** | 按评价、人气、案例质量排名 | 🔵 低优先级 | 需要一定设计师数量后才有效果 |
| **入驻激励/认证标识** | 认证设计师获得"已认证"蓝色标志 | 🟡 做完申请流程后 | 现在申请有但认证没有视觉反馈 |

## 三、业主/普通用户视角

| 功能 | 描述 | 优先级 | 理由 |
|---|---|---|---|
| **搜索对接真实数据** | 搜索 API → 搜索案例、文章、设计师、城市 | 🔴 高优先级 | 现在搜不出任何结果 |
| **案例评价摘要** | 案例详情页直接显示评分和评价摘要 | 🔴 高优先级 | 用户看案例最想看的就是别人评价 |
| **城市/地区浏览** | /city 页面：按城市筛选案例和设计师 | 🟡 中优先级 | 装修是强地域性需求 |
| **预算筛选** | 按预算区间筛选案例 | 🟡 做完真实数据搜索后 | 装修决策的核心维度 |

## 四、内容生态

| 功能 | 描述 | 优先级 | 理由 |
|---|---|---|---|
| **标签/话题系统** | 文章和案例加标签，支持按标签聚合浏览 | 🟡 中优先级 | 数据库已有 tags 字段，前端没有利用 |
| **问答列表页** | 把被写入 questions 表的问答展示出来 | 🔵 低优先级 | 有数据但没人能看 |
| **内容审核工作流** | 发布内容需要审核才可见 | 🔵 低优先级 | MVP 可以先相信用户 |
| **装修百科/攻略合集** | 把文章按话题组织成合集 | 🔵 长期规划 | 需要文章量够大 |

## 五、商业化潜力

| 功能 | 描述 | 优先级 | 理由 |
|---|---|---|---|
| **设计师入驻付费** | 入驻/发布案例达到一定数量后需付费 | 🔵 远期商业 | 先做流量再做变现 |
| **优选设计师（广告位）** | 首页推荐位/搜索结果前排展示付费设计师 | 🔵 远期商业 | 平台有流量后的自然变现方式 |
| **装修公司报价收集** | 用户发布需求 → 多家公司报价 | 🔵 独立产品方向 | 其实是另外一个产品了 |

## 六、技术完善

| 功能 | 描述 | 优先级 | 理由 |
|---|---|---|---|
| **Auth 打通业务系统** | 发布/评论/申请不再用 limit(1)，而是用真实登录用户 | 🔴 立即做 | 数据安全基础 |
| **Admin 鉴权** | admin 页面需要管理员角色才能访问 | 🟡 中优先级 | 现在任何人都能审核入驻申请 |
| **loading.tsx 全局加载态** | 每个路由加 loading.tsx | 🟡 高优先级做完后 | 无加载态的 SPA 体验不够好 |
| **not-found.tsx 404 页面** | 自定义 404 页面 | 🔵 低优先级 | 体验提升但非核心 |
| **评论表 migration** | 把 comments 表加到 migration SQL | 🟡 近期 | 防止下次重置数据库丢失表结构 |
| **真实图片上传** | 发布时上传真实图片 | 🟡 中优先级 | 有图才有说服力 |
| **PC 端响应式适配** | 让页面在大屏幕上也能正常显示 | 🟡 中优先级 | 用户不一定只在手机上看 |

## 🎯 建议路线图

### Sprint 1 — 核心体验闭环（立即做）
1. Auth 打通业务系统 🔴 — 发布/评论用真实用户
2. 点赞/收藏后端打通 🔴 — 最基础互动
3. 文章评论区 🔴 — 内容可讨论
4. 搜索 API + 对接真实数据 🔴 — 搜得到东西
5. 案例详情页加评价摘要 🔴 — 评价可见

### Sprint 2 — 设计师生态
6. Admin 鉴权保护 🟡
7. 设计师工作台 🟡 — 管理案例、看评价
8. 咨询消息打通 🟡 — 连接设计师和业主
9. 城市筛选页面 🟡 — 装修是地域性的
10. Tags 标签/话题系统 🟡

### Sprint 3 — 打磨体验
11. 浏览历史、个人资料编辑、PC 适配、loading.tsx、真实图片等
